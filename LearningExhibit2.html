Using Git Bash and Wget to Download Data

At this point I was excited about this exercise. I understood how useful being able to collect large amounts of data in this way could be for other projects I have worked on and for future projects. But, I did run into some
trouble with this right away.

It took a bit of back-and-forth to figure this out, but eventually I realized that my problem was with these jq commands in the .sh file. By changing these to /jq everything to seemed work just fine. Doing this provided the
"clean" and "dirty" files containing the ochin #s that I was not getting before.

It was immediately after this that I hit my next snag. I thought all was well until I reached the next part of the exercise dealing with the splitting of the output.txt file. The problem: I had no ouput file! So it was back
to playing with the code in the .sh file.

After reading each line very carefully I spotted the problem in line 38. I have a 64-bit machine, and had downloaded Wget64. By changing "wget" to "wget64," the problem was solved and I was now getting an output.txt
file.


Conclusion: A Very Important Exercise

First of all, this proved to be a very important exercise for me. I learned a little something about APIs, html, and the programs used for downloading data from a website. I also figured out that I did not have to run the .sh
file through Gitbash. All I actually had to do was double-click the .sh file and it ran automatically.
